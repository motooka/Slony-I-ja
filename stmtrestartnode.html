SLONIK RESTART NODE

Name 名称

RESTART NODE --  Restart Slony-I node Slony-I ノードの再起動

Synopsis 使用方法

RESTART NODE options;

Description 解説

Causes an eventually running replication daemon (slon process) on the specified node to shutdown and restart itself. Theoretically, this command should be obsolete. In practice, TCP timeouts can delay critical configuration changes to actually happen in the case where a former forwarding node failed and needs to be bypassed by subscribers.

指定されたノードで稼働しているレプリケーションのデーモン（slonのプロセス）を停止して再起動させます。理論上、このコマンドは使用するべきではありません。実際には、イベントを伝達しようとするノードが故障してサブスクライバによってバイパスする必要が発生した際に、TCPタイムアウトによって重要な設定変更イベントが遅延して伝達されることが起こりうるのです。

ID = ival

Node ID of the node to restart. 再起動させるノードのノードID

Example 例

  RESTART NODE ( ID = 2 );

Locking Behaviour ロック動作

No application-visible locking should take place. アプリケーションから観察可能なロックは獲得されません。

Version Information バージョン情報

This command was introduced in Slony-I 1.0; frequent use became unnecessary as of version 1.0.5. There are, however, occasional cases where it is necessary to interrupt a slon process, and this allows this to be scripted via slonik. このコマンドは Slony-I 1.0 で導入され、1.0.5 では頻繁に使用する必要はなくなりました。しかしながら、slonプロセスに割り込む必要は時々あるもので、このコマンドはその再起動という作業をslonikでスクリプト化するのに役立ちます。

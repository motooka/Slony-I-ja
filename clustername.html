SLONIK CLUSTER NAME

Name 名称

CLUSTER NAME --  preamble - identifying Slony-I cluster 事前処理 - slonikスクリプトが処理対象とする Slony-I のクラスタを特定します。

Synopsis 使用方法

CLUSTER NAME = [clustername;]

Description 解説

Must be the very first statement in every slonik script. It defines the namespace in which all Slony-I specific functions, procedures, tables and sequences are defined. The namespace name is built by prefixing the given string literal with an underscore. This namespace will be identical in all databases that participate in the same replication group. slonikスクリプトの中では、このコマンドが本当に一番始めに書かれていなければなりません。このコマンドは Slony-I の関数、プロシージャ、テーブル、シーケンスなどが所属する名前空間を定義します。名前空間の名前は、アンダースコアを含む文字列を接頭辞として付加して形成されます。この名前空間は同じレプリケーション・グループの中では全てのデータベースで同一です。

No user objects are supposed to live in this namespace, and the namespace is not allowed to exist prior to adding a database to the replication system. Thus, if you add a new node using pg_dump -s on a database that is already in the cluster of replicated databases, you will need to drop the namespace via the SQL command DROP SCHEMA _testcluster CASCADE; . この名前空間の中にはユーザのオブジェクトを格納してはなりませんし、レプリケーションに参加させる前に事前に作成していてもいけません。ですので、レプリケーションシステム内の既存のノードで pg_dump -s を使ってダンプを取得して新しいノードを作成する際は、この名前空間を DROP SCHEMA _testcluster CASCADE; によって削除する必要があるでしょう。

Example 例

     CLUSTER NAME = testcluster;

Version Information バージョン情報

This command was introduced in Slony-I 1.0 このコマンドは Slony-I 1.0 で導入されました。

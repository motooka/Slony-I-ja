Run-time Configuration ランタイムの設定

There are several configuration parameters that affect the behavior of the replication system. In this section, we describe how to set the slon daemon's configuration parameters; the following subsections discuss each parameter in detail. レプリケーションの動作に影響を与える設定項目がいくつかあります。この項では、slonの設定項目に値を設定する方法を説明します。以下、各項目について詳細な説明です。

All parameter names are case-insensitive. Every parameter takes a value of one of four types: boolean, integer, floating point, or string. Boolean values may be written as ON, OFF, FALSE, YES, NO, 1, 0 (all case-insensitive) or any unambiguous prefix of these. 全ての設定項目名はアルファベットの大文字小文字を区別しません。型は boolean, integer, 浮動小数点値, 文字列 のいずれかです。boolean型は ON / OFF / FALSE / YES / NO / 1 / 0 や、曖昧にならない範囲でのこれらの頭文字を利用することもできます。アルファベットの大文字小文字は区別されません。（訳注：恐らく TRUE も利用可能かと思われます。未検証。）

One parameter is specified per line. The equal sign between name and value is optional. Whitespace is insignificant and blank lines are ignored. Hash marks (#) introduce comments anywhere. Parameter values that are not simple identifiers or numbers must be single-quoted. １つの行には１つの設定項目を記載します。名前と値の間のイコール記号「=」は任意です。ホワイトスペース（半角スペース、タブ等）は意味を為しませんし、空の行は無視されます。シャープ記号「#」はコメントを表します。これはどこでも有効です（訳注：行頭でなくても構わない、の意味と思われます）。数字や単純な識別子でない値を設定する際は、シングルクォート「'」で囲わなければなりません。

Some options may be set through the Command-line, these options override any conflicting settings in the configuration file. いくつかの項目は起動時のコマンドラインでも設定することができ、設定ファイルのものと競合するようであれば、コマンドラインのものが優先されます。

1. Logging ロギング

syslog (integer)

Sets up logging to syslog. If this parameter is 1, messages go both to syslog and the standard output. A value of 2 sends output only to syslog (some messages will still go to the standard output/error). The default is 0, which means syslog is off. syslogへのロギングを設定します。「1」はメッセージを標準出力とsyslogの両方に送出します。「2」はsyslogのみに送出します（但しいくつかのメッセージは標準出力・標準エラー出力にも送出されます）。デフォルト値は「0」で、これはsyslogをオフにすることを意味します。

syslog_facility (string)

Sets the syslog "facility" to be used when syslog enabled. Valid values are LOCAL0, LOCAL1, LOCAL2, LOCAL3, LOCAL4, LOCAL5, LOCAL6, LOCAL7. The default is LOCAL0. syslogのファシリティ（ログの分類）を設定します。 LOCAL0, LOCAL1, LOCAL2, LOCAL3, LOCAL4, LOCAL5, LOCAL6, LOCAL7 を設定することができます。デフォルトは LOCAL0 です。

syslog_ident (string)

Sets the program name used to identify slon messages in syslog. The default is slon. syslogに送出されるメッセージ中のプログラム名を指定します。デフォルトは「slon」です。

log_level (integer)

Debug log level (higher value ==> more output). Range: [0,4], default 0 デバッグログのレベル（高ければ高いほどより多く出力されます）を設定します。0から4の範囲で指定します。デフォルトは0です。

There are nine log message types; using this option, some or all of the "debugging" levels may be left out of the slon logs. In Slony-I version 2, a lot of log message levels have been revised in an attempt to ensure the "interesting stuff" comes in at CONFIG/INFO levels, so that you could run at level 0, omitting all of the "DEBUG" messages, and still have meaningful contents in the logs. slonには9種類のメッセージが存在します。このオプションを指定すると、いくつかもしくは全てのデバッグログは、slonのログに出力されなくなります。Slony-Iバージョン2では、多くの「興味深い」ログが CONFIG/INFO レベルで出力されるように修正されたため、この設定値は0にしてデバッグログを出力しないようにしたとしても、十分に役立つログが出力されているようになります。

log_pid (boolean)

Determins, if you would like the pid of the (parent) slon process to appear in each log line entry. ログの各行に（親の）slonのPID（プロセスID）を出力するかどうかを設定します。

log_timestamp (boolean)

Determines if you would like the timestamp of the event being logged to appear in each log line entry. ログの各行にタイムスタンプを出力するかどうかを設定します。

Note that if syslog usage is configured, then this is ignored; it is assumed that syslog will be supplying timestamps, and timestamps are therefore suppressed. syslogの設定がなされている場合はこの設定は無視されることに気をつけて下さい。syslogがタイムスタンプを出力します。

log_timestamp_format (string)

A strftime()-conformant format string for use if log_timestamp is enabled. The default is "%Y-%m-%d %H:%M:%S %Z" log_timestamp が有効になっている際に使用される、strftime() 互換の日付フォーマット設定です。デフォルトは "%Y-%m-%d %H:%M:%S %Z" です。

pid_file (string)

Location and filename you would like for a file containing the Process ID of the slon process. The default is not defined in which case no file is written. slonプロセスが出力するプロセスIDファイルの場所とファイル名を設定します。デフォルトは未定義であり、PIDファイルは出力されません。
